# Plan: Sitemap Index URL Fix

## 1. Problem Statement

### Current Behavior

`SitemapService::collectUrl()` computes a URL by stripping the `OUTPUT_DIR` prefix from the
absolute `output_path` and appending the remainder directly to `SITE_BASE_URL`. The relevant
code (lines 51–54 of `SitemapService.php`) is:

```php
$relativePath = str_replace($outputDir . '/', '', $outputPath);
$loc = $siteUrl . '/' . ltrim($relativePath, '/');
```

This produces literal `.html` filenames in all URLs, including `index.html` files that web
servers serve as the directory root:

| `output_path` (after stripping OUTPUT_DIR) | Generated URL                              | Correct URL                          |
|--------------------------------------------|--------------------------------------------|--------------------------------------|
| `index.html`                               | `https://example.com/index.html`           | `https://example.com/`               |
| `podcast/index.html`                       | `https://example.com/podcast/index.html`   | `https://example.com/podcast/`       |
| `a/b/index.html`                           | `https://example.com/a/b/index.html`       | `https://example.com/a/b/`           |
| `guide/content-creation.html`              | `https://example.com/guide/content-creation.html` | `https://example.com/guide/content-creation.html` ✅ |

Search engines canonicalise `index.html` URLs to their parent directory form. Emitting the
wrong form in the sitemap causes search engines to treat the sitemap entry and the real
canonical URL as two separate resources, degrading crawl efficiency and potentially duplicating
pages in search indexes.

### Desired Behavior

Any `output_path` whose filename portion is exactly `index.html` must produce a trailing-slash
directory URL. All other filenames are left unchanged.

## 2. Proposed Code Change

### 2.1 Scope

Only two files change:

- `src/Features/Sitemap/Services/SitemapService.php` — add a private helper method and update
  `collectUrl()` to call it.
- `tests/Unit/Features/Sitemap/SitemapServiceTest.php` — add four new test methods covering
  the index-file cases.

No other file is touched. `Feature.php`, robots.txt, SEO audit commands, and all other
features are unaffected.

### 2.2 New Private Method: `normalizeUrl()`

Add the following private method to `SitemapService`. It encapsulates the `index.html`
rewrite logic in one place, keeping `collectUrl()` readable.

```php
/**
 * Rewrites index.html paths to canonical directory URLs.
 * "index.html"        → $siteUrl . '/'
 * "podcast/index.html" → $siteUrl . '/podcast/'
 * "a/b.html"          → $siteUrl . '/a/b.html'
 */
private function normalizeUrl(string $relativePath, string $siteUrl): string
{
    if ($relativePath === 'index.html') {
        return $siteUrl . '/';
    }

    if (basename($relativePath) === 'index.html') {
        return $siteUrl . '/' . rtrim(dirname($relativePath), '/') . '/';
    }

    return $siteUrl . '/' . ltrim($relativePath, '/');
}
```

### 2.3 Updated `collectUrl()` Method

Replace the single inline assignment on lines 51–54 with a call to `normalizeUrl()`. All
other logic (lastmod resolution, appending to `$this->urls`, returning `$parameters`) is
identical.

```php
$relativePath = str_replace($outputDir . '/', '', $outputPath);

// Construct canonical URL, rewriting index.html paths to directory URLs
$loc = $this->normalizeUrl($relativePath, $siteUrl);
```

### 2.4 Edge Cases Addressed

**Root `index.html` (no directory prefix).**
`$relativePath === 'index.html'` matches the first branch and returns `$siteUrl . '/'`.
PHP's `dirname('index.html')` returns `.`, so the first branch must exist to prevent the
second branch from producing `https://example.com/./`.

**Deeply nested `index.html` (e.g. `a/b/c/index.html`).**
`basename()` returns `index.html`; `dirname()` returns `a/b/c`; `rtrim` is a no-op on a
path with no trailing slash. Result: `https://example.com/a/b/c/`. Correct.

**Non-index HTML files (regression).**
Third branch returns `$siteUrl . '/' . ltrim($relativePath, '/')`, identical to the current
behaviour. No regression for normal content pages.

**Case sensitivity.**
The `basename($relativePath) === 'index.html'` comparison is case-sensitive. This is
intentional: `SitemapService` only consumes paths generated by this build tool, which
always produces lowercase filenames. `Index.html` or `INDEX.HTML` are not valid outputs
from the renderer and are explicitly out of scope.

**OUTPUT_DIR with a trailing slash.**
The existing `str_replace($outputDir . '/', '', $outputPath)` line would misfire if
`OUTPUT_DIR` already ends in `/`. This risk pre-dates the current fix and is unchanged by
it. Adding `rtrim($outputDir, '/')` as a defensive guard at that line would neutralise it
at zero cost, but it is out of scope for this fix. Flag for a follow-up ticket.

## 3. Test Coverage Plan

All changes go into `tests/Unit/Features/Sitemap/SitemapServiceTest.php`.

### 3.1 Existing Tests — No Changes Required

| Method | Status |
|--------|--------|
| `testCollectUrl()` | Uses `foo/bar.html`; asserts only that parameters are returned unchanged. Passes. |
| `testCollectUrlSkipsIfNoOutputPath()` | No URL generation. Passes. |
| `testGenerateSitemap()` | Uses `foo/bar.html`; asserts `<loc>https://example.com/foo/bar.html</loc>`. Third branch of `normalizeUrl()` produces identical result. Passes. |
| `testGenerateSitemapSkipsIfNoUrls()` | No URL generation. Passes. |

### 3.2 New Test Methods to Add

Add these four methods after the last existing test method.

**Test 1 — Root `index.html` → trailing slash on base URL**

```php
public function testCollectUrlRootIndexHtmlProducesTrailingSlash(): void
{
    $parameters = [
        'output_path' => $this->tempDir . '/index.html',
        'metadata'    => ['date' => '2024-01-15'],
    ];
    $this->service->collectUrl($this->container, $parameters);
    $this->service->generateSitemap($this->container, []);

    $content = file_get_contents($this->tempDir . '/sitemap.xml');
    $this->assertStringContainsString('<loc>https://example.com/</loc>', $content);
}
```

**Test 2 — Single-level subdirectory `index.html`**

```php
public function testCollectUrlSubdirectoryIndexHtmlProducesDirectoryUrl(): void
{
    $parameters = [
        'output_path' => $this->tempDir . '/podcast/index.html',
        'metadata'    => ['date' => '2024-02-01'],
    ];
    $this->service->collectUrl($this->container, $parameters);
    $this->service->generateSitemap($this->container, []);

    $content = file_get_contents($this->tempDir . '/sitemap.xml');
    $this->assertStringContainsString('<loc>https://example.com/podcast/</loc>', $content);
}
```

**Test 3 — Nested `index.html` (two levels deep)**

```php
public function testCollectUrlNestedIndexHtmlProducesNestedDirectoryUrl(): void
{
    $parameters = [
        'output_path' => $this->tempDir . '/a/b/index.html',
```

**Test 4 — Regular HTML file is unchanged (regression guard)**

```php
public function testCollectUrlRegularHtmlFileIsUnchanged(): void
{
    $parameters = [
        'output_path' => $this->tempDir . '/guide/content-creation.html',
```

## 4. Implementation Steps

1. Open `src/Features/Sitemap/Services/SitemapService.php`.
2. Add the `normalizeUrl()` private method immediately after `collectUrl()` and before `generateSitemap()`.
3. In `collectUrl()`, replace:
   ```php
   $loc = $siteUrl . '/' . ltrim($relativePath, '/');
   ```
   with:
   ```php
   $loc = $this->normalizeUrl($relativePath, $siteUrl);
   ```
4. Open `tests/Unit/Features/Sitemap/SitemapServiceTest.php`.
5. Append the four new test methods from Section 3.2 after `testGenerateSitemapSkipsIfNoUrls()`.

7. Run the full unit suite: `lando phpunit tests/Unit/`
